Southbrook Estimating

A Prodboard-class kitchen Order Builder on Odoo 19 Community Edition

Sales reps and dealers configure full kitchens line-by-line in a multi-zone Order Builder. Every selection re-prices live through the channel-resolved pricelist; every confirmed order materialises a parametric BoM and an analytics row for downstream forecasting.

What ships in this addon

Order Builder backend

  • Multi-zone grid: BASE_RUN, WALL, TALL, ISLAND, ACCESSORY, OTHER with free-text label
  • Channel-driven pricelist resolver: 6 channels (Retail, Dealer, Tradesperson with 3 tiers, KD, Big-Box, Refacing CTHS)
  • "Duplicate as Draft" for iterative-design workflows (Image Floor pattern); parent_order chain + version tracking
  • Live re-pricing on partner switch
  • Illustrative-seed banner makes demo data unmistakable

Parametric BoM + Reports

  • Cabinet panel-cut math from W × H × D + family + door count via _compute_panel_dimensions
  • Maple-box +2-week lead-time bump folds into produce_delay via a dedicated lead_time_extra attribute field
  • Signature Spec Sheet (customer-facing PDF, sale.order-bound)
  • Shop Copy (MO-companion, mrp.production-bound)
  • Door Order (per-SO door schedule)
  • Auto-captured analytics on order confirm: channel, series, cabinet/panel/door counts, lifecycle timestamps

Declarative configurator rules

Four business rules are encoded as product.config.line records in data/config_rules.xml — never as Python if branches. Adding a tier or a series is a data edit.

RuleBehaviour
Series → door styleContractor exposes slab only; Elegance exposes 5-piece only
Box material → seriesMaple available only on Contemporary and Elegance; standard on Signature
Width → door count9–21 in → 1-door; 24–36 in → 2-door (cascades into hinge / handle / door BoM lines)
Family → soft-closeCorner bi-fold hides the soft-close option entirely

Dependencies

This addon depends on the OCA product_configurator suite, which is not on the Odoo Apps Store. Install the four OCA modules from github.com/OCA/product-configurator before installing this addon:

  • product_configurator
  • product_configurator_mrp
  • product_configurator_sale
  • website_product_configurator (only required by the companion website addon, not this one)

Plus the Odoo Community core: mrp, sale_management, stock, account, contacts, crm.

Scope

This is Phase 1 of a 4-phase build. Phase 2 ships the customer-facing one-page configurator with 2D rendering; Phase 3 adds the Three.js procedural 3D layer; Phase 4 adds the Accucutt cut-list bridge. Phase 1 alone is enough for the sales-rep Order Builder to be operationally useful.

The build is open-source under LGPL-3 and authored for Southbrook Cabinetry; the codebase is documented forensically through PUNCHLIST.md (Q-numbered locked decisions + NF-numbered surfacings) and docs/PHASE_1_RETROSPECTIVE.md (process patterns + design discipline).

Installation

# 1. Install OCA product_configurator suite (see Dependencies section)
# 2. Drop southbrook_estimating into your Odoo addons path
# 3. Install on a fresh 19.0 CE database:
odoo-bin -d sb_phase1 -i southbrook_estimating --stop-after-init

# With demo data:
odoo-bin -d sb_phase1_demo -i southbrook_estimating --demo \
         --stop-after-init

# Run the test suite:
odoo-bin -d sb_phase1_test -i southbrook_estimating --demo \
         --test-enable --stop-after-init --test-tags=southbrook

Quality

  • 95 tests across 13 files with a 10-step Phase-1 smoke gate
  • Custom-code surface bounded to 7 routines by Build Spec § 4 discipline; data files + OCA _inherit handle the rest
  • Pre-commit XML lint (scripts/lint-xml.sh) catches well-formedness errors before they reach module install
  • Every decision tracked through Q1–Q23 + NF1–NF15 in PUNCHLIST, cited by Q/NF number in every commit message body
